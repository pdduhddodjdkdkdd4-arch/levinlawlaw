## 数据持久化

### 重要说明

由于是纯静态部署，无法直接读写服务器文件，采用以下方案：

#### 方案1：使用浏览器存储（推荐）
- 数据保存在用户浏览器的 localStorage 中
- 支持CSV导入/导出功能
- 适合单机使用场景

#### 方案2：手动管理CSV文件
1. 定期从管理后台导出CSV文件
2. 将CSV文件保存在本地或服务器
3. 需要时通过管理后台的"导入CSV"功能恢复数据

### CSV文件格式

CSV文件格式如下（包含表头）：
```
ID,姓名,邮箱,电话,平台,金额,提交时间,IP
1,张三,test@example.com,13800138000,Example平台,$1000,2024-01-01 12:00:00,127.0.0.1
```

## 使用流程

### 用户端
1. 访问 `contact-us.html`
2. 填写表单信息
3. 点击提交
4. 查看成功提示和提交的信息

### 管理端

#### 登录
1. 访问 `login.html`
2. 输入用户名和密码
3. 点击登录进入管理后台

#### 数据管理
1. **查看数据**：在表格中浏览所有提交记录
2. **筛选数据**：使用顶部筛选器按条件过滤
3. **编辑数据**：点击"编辑"图标修改记录
4. **删除数据**：点击"删除"图标删除单条记录
5. **批量操作**：勾选记录后进行批量导出

#### 数据导出/导入
1. **导出选中**：勾选记录后点击"导出选中"
2. **导出全部**：点击"导出全部"下载完整CSV
3. **导入数据**：点击"导入CSV"选择文件导入

#### 文件名格式
导出的文件命名格式：`YYYYMMDDHHMMSS_levinlaw.csv`

## 部署说明

### 静态部署
将整个项目文件夹上传到任意静态托管服务：
- Nginx
- Apache
- Netlify
- Vercel
- GitHub Pages
- 虚拟主机

### 重要注意事项
1. 数据存储在浏览器端，不同浏览器/设备不互通
2. 建议定期导出CSV文件备份
3. 管理后台通过隐蔽路由保护，但无服务器端安全机制
4. 生产环境建议添加额外的安全措施

## 安全建议

1. **路由保护**：`admin_26_5_9.html` 是隐蔽路由，请保密
2. **定期备份**：定期导出CSV文件
3. **密码安全**：可在 `js/csv-manager.js` 中修改默认密码
4. **访问控制**：生产环境建议添加HTTP基础认证

## 常见问题

### Q: 数据会丢失吗？
A: 数据保存在浏览器本地存储中，清除浏览器数据会导致数据丢失。建议定期导出备份。

### Q: 可以在多台设备间同步数据吗？
A: 可以通过CSV导入导出功能在设备间转移数据。

### Q: 修改登录密码？
A: 编辑 `js/csv-manager.js` 文件，修改 `AuthManager` 类中的 `CREDENTIALS` 对象。

### Q: CSV文件损坏了怎么办？
A: 只要浏览器没清除数据，就可以从管理后台重新导出。

## 技术支持

如有问题，请检查：
1. 浏览器控制台是否有错误信息
2. 浏览器是否启用了localStorage
3. 文件路径是否正确
